Существующие потенциальные проблемные места:
1. Долгие процесс расчета стоимости (до 30 минут для сложных моделей).
2. При увеличении нагрузки система долго не отвечает.
3. Используемый брокер сообщений RabbitMQ не является оптимальным решением для текущей системы.
4. Долгий ответ системы на чтение данных из БД 
5. Текущий CI/CD по описанию предполагает только автоматическое тестирование (CI). Ручная доставка сборки конечным пользователям вызывает задержки + потенциальные ошибки при сборке.


Необходимые инициативы для устранения нежелательных ситуаций:
1. Оптимизация backend системы MES (исходный код доступен, в команде есть backend инженер и тимлид c#).
2. Добавить дополнительные инстансы backend приложения MES 
3. Заменить брокер сообщения RabbitMQ на Apache Kafka как более подходящего с точки зрения масштабируемости системы.
4. Добавить кэширование БД для быстрого чтения данных на стороне пользователя.
5. Автоматизировать процессы развертывания и тестирования.
