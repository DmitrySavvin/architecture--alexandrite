Существующие потенциальные проблемные места:
1. Долгий процесс расчета стоимости (до 30 минут для сложных моделей) при отсутствии инструмента контроля состояния. В случае возникновения ошибки имеется риск потери результатов расчета без возможности определения проблемного места в системе.
2. При увеличении нагрузки система может долго не отвечать. Дальнейшее увеличение нагрузки может привести к еще более длительному отклику системы, что в свою очередь может повлечь блокировку запросов по таймаутам.
3. Долгий ответ системы на чтение данных из БД. С учетом увеличения нагрузки нагрузка на БД будет повышаться.
4. Текущий CI/CD по описанию предполагает только автоматическое тестирование (CI). Ручная доставка сборки конечным пользователям вызывает задержки + потенциальные ошибки при сборке.


Необходимые инициативы для устранения нежелательных ситуаций (расставлены в порядке приоритета реализации):
1. Повышение наблюдаемости системы путем внедрения мониторинга, логирования и трейсинга. При реализации данного пункта появятся удобные инструменты для своевременного выявления и устранения дефектов системы. 
2. Внедрить кэширование для повышения отказоустойчивости и снижения загрузки системы. 
3. Добавить дополнительные инстансы backend приложения MES. Внедрить шардирование базы данных для ускорения запросов к базе данных.
4. Автоматизировать процессы развертывания и тестирования.
5. Оптимизация backend системы MES (исходный код доступен, в команде есть backend инженер и тимлид c#). 


С учетом имеющейся команды и возможности расширения в ближайшие полгода возможна реализация первых трех пунктов. Данные пункты определены наиболее приоритетными, поскольку позволят повысить степень наблюдаемости системы и снизить общую нагрузку на систему. При наличии инструментов мониторинга, логирования и трейсинга устранение дефектов будет занимать значительно меньшее время чем при отсутствии данных инструментов.
